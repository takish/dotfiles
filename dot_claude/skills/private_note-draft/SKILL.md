# note.com記事本文生成

指定されたアウトラインから、完成度の高い記事本文を生成します。

## 入力

アウトラインファイルのパス

例: `20251221-outline-limiter.md` または `80_note/03_Drafts/20251221-outline-limiter.md`

## 執筆ルール

### 文章スタイル
| 項目 | ルール |
|------|--------|
| 文体 | です・ます調で統一 |
| トーン | 親しみやすく、専門性を感じさせる |
| 1段落 | 3文以内 |
| 専門用語 | 必ず（）で補足説明 |

### 構造
- **文字数**: 7000-12000字
- **リード文**: 300-400字、読者の課題に共感
- **各H2セクション**: 800-1200字
- **まとめ**: 200-300字、明確なCTA

### AIEO最適化（必須）
- 結論ファースト
- 主語を省略しない
- 曖昧表現禁止（〜と思われる、〜かもしれない）
- 見出しは断定形
- 箇条書き多用

## 画像挿入指示

記事中に画像が必要な箇所にはコメントで指示:

```html
<!-- 画像: [画像の説明] -->
```

## 出力形式（note-post-mcp互換）

```markdown
---
title: [最終タイトル]
tags:
  - [タグ1]
  - [タグ2]
  - [タグ3]
type: draft
platform: note
status: draft
source_outline: [元アウトラインのパス]
created: YYYY-MM-DD
---

[リード文]

## [H2見出し1]

[本文...]

### [H3見出し1-1]

[本文...]

## まとめ

[要点整理と次のステップ]

---
## 編集メモ（投稿前に削除）
- 推定文字数: XXXX字
- 画像挿入箇所: N箇所
- 要確認事項: [レビューポイント]
```

## 出力先

ファイルを `80_note/03_Drafts/` に保存。
ファイル名: `[日付]-draft-[タイトル略称].md`

完了後、`80_note/04_Review/` への移動を提案。
