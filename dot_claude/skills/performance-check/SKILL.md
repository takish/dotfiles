# Performance Check

現在の実装のパフォーマンスを分析し、改善点を提案します。

## 分析観点

詳細なチェックリストは [references/checklist.md](references/checklist.md) を参照。

### 主要な確認項目
1. **フロントエンド**: 再レンダリング、メモ化、バンドルサイズ
2. **バックエンド**: クエリ最適化、キャッシュ、非同期処理
3. **共通**: アルゴリズム計算量、リソース解放

## 出力形式

改善案は、実装の難易度と効果のバランスを考慮して提案：

```markdown
### 高効果・低コスト
[すぐに実施すべき改善]

### 高効果・高コスト
[計画的に取り組むべき改善]

### 低効果
[余裕があれば対応]
```
