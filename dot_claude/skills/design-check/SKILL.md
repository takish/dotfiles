# Design Check

現在の実装のUI/UXデザインをチェックします。

## チェック観点

詳細なチェックリストは [references/checklist.md](references/checklist.md) を参照。

### 主要な確認項目
1. **一貫性**: デザインシステムとの整合性、他の画面との統一感
2. **ユーザビリティ**: 操作の直感性、フィードバック
3. **アクセシビリティ**: キーボード操作、スクリーンリーダー対応
4. **レスポンシブ対応**: 各ブレークポイントでの表示

## 出力形式

改善案は、実装の優先順位と共に提示：

```markdown
### 優先度: 高
[ユーザー体験に直接影響する問題]

### 優先度: 中
[改善が望ましい問題]

### 優先度: 低
[細かい調整]
```
