# Code Review

指定されたコードに対してレビューを実施し、問題点と改善案を提示します。

## レビュー観点

詳細なチェックリストは [references/checklist.md](references/checklist.md) を参照。

### 主要な確認項目
1. **DRY原則**: 重複コードの有無、適切な抽象化
2. **ベストプラクティス**: 言語・フレームワーク固有のイディオム
3. **潜在的な問題**: バグの可能性、エッジケースの考慮漏れ

## 出力形式

各問題点について以下を含めること：
- 問題の説明
- 具体的な改善案
- 優先度（高/中/低）

```markdown
### 🔴 Critical（高）
[必ず修正が必要な問題]

### 🟡 Warning（中）
[修正を推奨する問題]

### 🟢 Suggestion（低）
[あれば良い改善]
```
