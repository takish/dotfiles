# Code Review Checklist

## 1. DRY原則
- 複雑度：関数やメソッドが適切な大きさに分割されているか
- 重複コードが存在しないか
- 共通処理が適切に抽象化されているか

## 2. ベストプラクティス
- 言語・フレームワーク固有のイディオムに従っているか
- エラーハンドリングは適切か
- ログ出力は適切なレベルで行われているか
- 命名規則は一貫しているか
- コメントは必要十分か

## 3. 潜在的な問題
- バグの可能性がある箇所
- エッジケースの考慮漏れ
- 型の不整合やnull参照の危険性
- リソースリークの可能性
- 競合状態の可能性

## 4. セキュリティ
- 入力値の検証
- 認証・認可の適切性
- 機密情報の取り扱い

## 5. パフォーマンス
- 不要なループや計算
- N+1問題
- メモリ使用量
